Setting up Node.js & Express.js backend

Next we need to set up our backend, we'll be using Node.js and Express.js

Let's start to setup our file structure and create a folder called movie-reviews, and inside movie-reviews create another folder called 'backend'

After running npm init inside of backend, npm creates a package.json for us, which contains meta-data about our Node.js project

Next inside our backend folder, run the following in order to install the dependencies we need

npm install express cors mongodb dotenv

After running, you'll notice in our package.json a new property 'dependencies' has been created

Express

Express is a framewok that acts as a layer atop Node.js web server making it much easier to develop web applications, it simplifies the API's of Node.js and adds helpful features, and assists us in organizing our apps web functionality with middleware, routing and many others

CORS

CORS or Cross-Origin Resource Sharing, by default modern browswers won't allow frontend clients to talk to REST API's

The broswer will by default block the requests sent from client to server as a security mechanism, to make sure that the client-side browser JS code can ONLY talk to their own allowed server, not other servers that could potentially run mailcious code

To circumvent this security mechanism, we can enable CORS checking, which will use additional HTTP headers to tell browsers to give a web app running at one origin, access to a specified resources at another

The CORS package we installed earlier provides an Express.js middleware than can enable CORS with different options so we can make the correct connections on the network

The mongodb package we installed allows us to interact with our MongoDB database

And finally the dotenv loads environment variables from tue process.env file instead of setting environment variables on our development machine, which simplifies development work, we'll go into process.env in greater detail later

What are dependencies anyway?

Well dependencies contain the dependency packages, and their specific version number like:

"cors": "^2.8.5",

Every single time we use 'npm install yadayada' npm will save and keep track of all the yadayadas and their version everytime

How it works is that npm, when we use npm install, it will fetch the latest versions of what we want and auto-saves it in that folder called 'node_modules'
